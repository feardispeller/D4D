Architecture diagnostics

Five checks decide how a sealed tape travels and whether it can play forever, offline. Run them on the phone the gift is for. Each result is a real go / no-go.

Environment00

Before anything touches the network: is this page in a real browser context where external requests are allowed? If it's sandboxed inside a preview, song search physically can't work — and the fix is to open the file in Safari.

checking…
Song search & previews01

Can we find any song and pull its 30-second preview + cover art, with no API key and no CORS wall? (JSONP against the iTunes Search API.)

Live preview playback02

Plays the selected preview through an <audio> element. This is the guaranteed path — works cross-origin without CORS. The floor we can always stand on.

Embed audio for a forever-tape03

The crux. Can we fetch the preview's bytes and decode them? If yes → we bake the audio into the gift so it plays offline forever, like a real cassette, and we get Web Audio crossfades. If no → the gift needs internet, or we add full-track upload for permanence.

Photo compression budget04

Pick a real photo from this phone. We resize to 1600px / JPEG and measure the size, so we know how many photos fit in a giftable tape file.

iOS audio unlock05

The recurring Safari trap: audio only starts inside a real tap. This confirms our "press play to begin" gesture wakes the engine. You should hear one short tone.
